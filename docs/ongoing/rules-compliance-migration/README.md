# ルール準拠への移行タスク

## 概要

このタスクは、プロジェクトのコードを `.cursor/rules/` 配下のコーディング規約に準拠させることを目的としています。

## 関連ドキュメント

- `WORKING_GUIDE.md` - **作業ガイド（必読）**: 作業方針と手順の詳細
- `validation-report.md` - ルールとコードの整合性検証結果の詳細
- `migration-plan.md` - ルール準拠への移行計画
- `checklist.md` - 移行作業のチェックリスト

## 作業方針

### 進め方
1. **フェーズごとに順次実施**: `migration-plan.md` に記載されたフェーズを順番に進める
2. **フェーズごとにPR作成**: 各フェーズが完了したら、`.cursor/rules/create-pullrequest.mdc` の手順に従ってPRを作成
3. **不明点は適宜質問**: 作業中に不明点があれば、都度確認する

### PR作成のタイミング
- 各フェーズの作業が完了した時点でPRを作成
- PRはDraftで作成（指示がない限り）
- 関連Issueがある場合は必ずリンクを記載
- Issueがない場合は「Issueなし」と明記

### 作業の流れ
1. フェーズの作業を実施
2. テストを実行して動作確認
3. チェックリストを更新
4. PRを作成
5. 次のフェーズに進む

詳細は `migration-plan.md` と `checklist.md` を参照してください。

## 主な作業内容

1. ルールとコードの整合性検証
2. ドキュメンテーション（Yard記法）の追加
3. カスタムバリデーションの分離
4. 認可ロジックの分離（Pundit導入）
5. その他の規約準拠

詳細は各ドキュメントを参照してください。
